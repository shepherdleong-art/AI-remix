# 三期·批量生成工作流规划 V1.1

> 日期：2026-07-20
> 状态：需求已锁定（13 项决策 + 7 项复盘优化），待动工
> 前置：二期时间线功能已验收（见 PHASE2_TIMELINE_PLAN_V1.md、AI_HANDOFF_2026-07-17.md）
> 原则：**单条流程一行不动**；批量为独立模式，底层最大复用。

---

## 1. 背景与目标

用户场景：电商矩阵号铺量，需要把一批素材（8-80 条、单条 ≤10s，AI 素材 10 几 M / 实拍 ≤50M 偶 100M）
混剪成多条 ~15s 成片，发往电商平台。**核心诉求是躲避平台查重算法**——
不仅素材层面少重复，成片整体的素材组合、排列顺序、封面、标题、BGM、文件名都要差异化。

用户目标：
- 单次上传 8-80 条素材，一次性并发分析
- 脚本界面批量填入 N 个脚本
- 系统按目标成片数量"科学调用素材"：语义匹配优先，最大化减少素材重复，均匀消耗全部素材
- 抽查式人工干预（重点是实拍素材的入出点修整）
- 卡片队列式批量管理界面，简单可靠易用
- 整体生成效率提升

## 2. 需求决策记录（2026-07-20 问答锁定）

| # | 决策点 | 结论 |
|---|--------|------|
| D1 | 数量映射 | 一一对应为主：N 脚本 → N 成片；支持一脚本多片裂变（用户可选每脚本出几条），素材消耗科学均衡 |
| D2 | 重复约束 | 软约束 + 智能降级：先全不重复；素材不足时优先重复高语义匹配素材，并通过换入点/换区间/换排列组合制造差异 |
| D3 | 分析策略 | 上传即全量并发分析；并发数做成可调参数（默认 5，用户确认供应商限额后可调 10），带 429 自动降速重试 |
| D4 | 分析缓存 | 文件哈希 → 分析结果 JSON 永久缓存（每条几 KB），跨批次/跨项目复用，同文件永不重复分析 |
| D5 | 人工干预 | 抽查式：自动生成全部初剪 → 卡片队列扫片 → 不满意才进入微调（重点：实拍素材 Trim 入出点） |
| D6 | 界面形态 | 卡片队列为主骨架 + 抽屉式单条编辑；不用浏览器标签页（10-30 条时标签页失控） |
| D7 | TTS / 字幕 | 全批次音色/语速/音量统一，设一次；字幕样式（字体/颜色/位置）全局统一，设一次 |
| D8 | 封面 | 每条独立可调、封面帧与标题互不相同（查重对抗）；系统默认自动差异化（选不重复帧 + 标题变体 + 模板轮替），用户零操作可用、想改随时改 |
| D9 | 入口形态 | 双模式：首页/新建时选"单条精细"或"批量生产"；单条四步流程保持不变 |
| D10 | 导出 | 串行后台队列 + 全部完成通知；期间可继续编辑未导出卡片；可暂停/取消单条；不做并行导出（收益低且卡预览） |
| D11 | 素材沉淀 | 素材随批次走，视频文件不入库；仅保留分析缓存；本地文件夹挂载式素材库放二期增强 |
| D12 | 批次持久化 | 完整批次持久化（素材引用、分配方案、每条成片编辑状态、导出进度），断点续作；历史记录支持勾选增删；**一律存相对路径**（换电脑/换盘不炸，见 O6） |
| D13 | BGM | 批次内系统自动从音乐库轮替分配**不重复**曲目（音量/淡出统一处理）；每条成片保留人工更换单条音乐的入口 |

## 3. 用户流程与界面设计（批量模式）

### 阶段 0 · 入口
- 首页/新建项目时二选一：【单条精细】（现有流程）/【批量生产】（新）
- 批量项目在历史列表中有类型标识，点入恢复到离开时的阶段与状态

### 阶段 1 · 素材上传与分析
- 拖拽/多选上传 8-80 条素材；列表显示缩略图、时长、大小、分析状态
- **上传去重**：同文件（快哈希）重复拖入自动忽略并提示（O4）
- 上传完成自动开始并发分析（可调并发，默认 5；429 自动降速重试）
- 命中哈希缓存的素材秒过（标注"已缓存"）
- 顶部总进度条；失败的素材可单独重试，不阻塞整体
- 上传时可继续下一步准备（脚本录入可先行）

### 阶段 1.5 · 素材预修（O1，实拍批次的关键环节）
- 素材逐条过：内嵌播放器 scrub，标记可用区间（入点/出点）
- 键盘快捷键流（如 I/O 打点、Enter 下一条），80 条几分钟过完
- AI 自动建议头尾裁剪点（黑帧/静默/静止帧检测），人工一键确认或微调
- AI 素材（生成质量稳定）可整批跳过此环节
- 预修结果写入素材记录，阶段 3 分配**只在可用窗口内取段**

### 阶段 2 · 脚本批量录入
- 多脚本编辑器：左侧脚本列表（可增删、拖拽排序），右侧当前脚本文本
- 支持一键粘贴批量导入（空行/分隔符切分）
- 每个脚本可设置"出片数量"（默认 1，即 D1 的裂变开关）
- 全局设置区：音色/语速/音量（一次）、字幕样式（一次）、目标总时长档位（沿用 15s）、BGM 池（默认全库轮替，可圈选范围）
- **可行性预估实时条**（O2）：随素材量/可用区间/脚本数实时显示"建议 ≤ X 条不重复成片"，超出时黄字预警但不强制拦截
- "开始分配"按钮 → 进入智能分配

### 阶段 3 · 智能分配（核心新增）
- 系统自动完成：TTS 批量预生成（得槽长）→ 全局素材分配（只在预修可用窗口内取段）→ BGM 轮替分配 → 封面自动差异化 → 每条成片初剪拼装
- 分配过程可见：进度 + 完成后出"分配报告"（每素材使用次数分布、重复率、BGM 分配表、预警）
- 素材不足时按 D2 降级，并在报告中明确标出"哪些成片用了重复素材、重复在哪几段"
- **脚本再编辑规则**（O3）：此后改脚本只使该片失效并标"待重新分配"，用户确认后单独重跑该片

### 阶段 4 · 卡片队列审改（批量主界面）
- 每条成片一张卡片：内嵌小预览（静音循环，悬停 scrub）、脚本名、状态、时长、封面缩略图
- 状态机：待生成 / 生成中 / 待确认 / 待重新分配 / 已确认 / 导出中 / 已完成 / 失败
- 状态着色 + 顶部筛选（如"只看待确认"）
- **相似度预警**：批次内两成片素材重合度超阈值 → 卡片标黄，点击看"和哪几条撞、撞在哪几段"
- 点卡片 → 抽屉式（右侧滑出）浓缩编辑器：
  - 素材序列列表 + 每段 Trim（复用 TrimEditor / 重选时段交互）
  - 单段替换（保留原入点设置）
  - 字幕校对、封面编辑（换帧/改标题，复用 CoverDrawer 能力）
  - **单条 BGM 更换入口**（D13）
  - 改完返回，卡片预览自动刷新、位置不跳
- 已确认/已导出卡片锁定，改动需先解锁
- 封面默认已自动差异化（D8），用户只改不满意的

### 阶段 5 · 批量导出
- "导出已确认" / "全部导出" → 串行后台队列
- 队列面板：每条进度、可暂停/取消单条、失败可重试
- 期间可继续编辑未导出卡片（队列动态追加）
- 全部完成：本地通知 + 声音提醒
- 输出目录按批次归档（批次名/日期子文件夹）
- **文件命名**：`批次名_序号_标题变体.mp4`（O5）

### 4.2 扩展：多成片联合分配求解器（match_solver 升级）

现状：match_solver 已有 `usage_count` 素材使用计数与不变量校验（单成片维度）。
扩展为**批次级联合分配**：

```
输入：素材池 M 条（含语义向量/场景标签/预修可用区间 [usable_in, usable_out]）、
      脚本 N 个（含裂变数 k_i）、每片 TTS 槽长（每片时长 = TTS 槽长，不变量保持）
输出：N' = Σk_i 条成片的素材分配方案（每段：素材 id + 入点/出点，全部落在可用窗口内）

目标函数（按优先级）：
  1. 语义匹配度最大化（沿用现有匹配分）
  2. 素材使用均衡：usage_count 方差最小（科学平均消耗，D1）
  3. 重复惩罚：同素材跨成片复用扣分；同素材+同区间复用重扣分
  4. 组合差异：成片两两的素材集合 Jaccard 相似度低于阈值（查重对抗）

降级策略（素材不足时，D2）：
  - 轮次制分配：第一轮每条素材最多用 1 次，第二轮放宽到 2 次……直到满足片数
  - 重复时优先选"语义匹配分最高"的素材
  - 重复素材强制换区间（在可用窗口内取与上次不重叠的子段）
  - 同素材进入同批次多片时，排列位置强制错开
```

- 求解器输出带"分配报告"数据结构（使用率分布、重复明细、相似度矩阵、BGM 分配表），供阶段 3/4 展示
- 每条成片的时长不变量（= TTS 槽长）在批量下逐片独立成立，校验逻辑复用
- 可行性预估（O2）= 求解器的轻量预跑：Σ可用窗口时长 ÷ 每段均长 ÷ 每条片段数 → 建议最大片数

## 5. 查重对抗策略（贯穿设计）

| 层面 | 手段 |
|------|------|
| 素材组合 | 联合分配最小化跨片重复；Jaccard 相似度监控 + 卡片预警 |

| 标题 | AI 生成 N 个措辞变体或用户标题池轮替 |
| 音频 | BGM 批次内不重复轮替（D13） |
| 文件名 | 导出命名含标题变体（O5） |
| 可见预警 | 分配报告 + 卡片标黄，把判重风险在导出前暴露给用户 |

## 6. 效率优化清单

1. 分析并发化 + 哈希缓存（最大头：80 条分析从串行几十分钟 → 并发后几分钟，缓存命中秒过）
2. 大文件快哈希（首尾采样），上传即去重不卡盘（O4）
3. TTS 批量预生成（分配前一次跑完，避免逐片等待）
4. 预览 assemble md5 缓存跨成片复用（同素材同区间片段只渲染一次）
5. 导出串行队列后台跑，不阻塞编辑
6. 卡片预览用低码率代理缩略（悬停 scrub 流畅）
7. 分配求解为本地计算，秒级

## 9. 验收标准（Batch-P1）

1. 80 条素材上传→分析完成，缓存全命中时二次上传秒过；重复拖入同文件被去重
2. 素材预修：AI 头尾建议可一键确认，手工标区间后分配不越界
3. 30 脚本 → 30 条初剪自动生成，分配报告数据正确；可行性预估条数字准确
4. 素材不足场景按 D2 降级且报告明示重复明细
5. 改脚本只使对应成片失效并可单独重跑，其余成片不动
6. 卡片队列扫片流畅（低码率预览），抽屉编辑 Trim/替换后预览=导出（铁律不破）
7. 批次关掉重开完整恢复（含换机路径场景）；历史可勾选增删
8. 30 条串行导出后台完成，期间编辑不卡，完成有通知；文件名符合 O5 规范
9. 批次内封面两两不同帧、标题两两不同、BGM 两两不同（库容量足够时）
10. 相似度超阈值成片被标黄且可查看撞车明细
11. 单条四步流程回归无损
